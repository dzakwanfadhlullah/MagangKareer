## PHASE 1: NAVBAR ENHANCEMENT
**Fokus:** Interaksi, Responsif Mobile, dan Polish Visual

### 1.1 Mobile Hamburger Menu dengan Animasi Slide
**Masalah:** Navbar saat ini menyembunyikan navigasi di mobile (`hidden md:flex`), tidak ada menu hamburger.

**Solusi:**
- Tambahkan komponen `MobileMenu` dengan hamburger icon (Menu/X toggle)
- Implementasi slide-in drawer dari kanan menggunakan Framer Motion
- Menu overlay dengan backdrop blur dan close-on-click-outside
- Animasi stagger untuk setiap menu item

**File yang dimodifikasi:**
- `Navbar.tsx` - Tambah hamburger button dan mobile menu state
- `MobileMenu.tsx` [NEW] - Komponen drawer terpisah

---

### 1.2 Scroll-Aware Navbar dengan Shadow Dinamis
**Masalah:** Navbar tidak bereaksi terhadap scroll, shadow/blur tetap sama.

**Solusi:**
- Gunakan `useScroll` dan `useMotionValueEvent` dari Framer Motion
- Saat scroll > 50px: tambah shadow-md, tingkatkan opacity background ke 90%
- Saat scroll = 0: kembali ke state transparan dengan blur ringan
- Transisi smooth 200ms untuk semua perubahan

**Kode contoh:**
```tsx
const { scrollY } = useScroll();
const [scrolled, setScrolled] = useState(false);

useMotionValueEvent(scrollY, "change", (latest) => {
  setScrolled(latest > 50);
});
```

---

### 1.3 Active Link Indicator dengan Intersection Observer
**Masalah:** User tidak tahu section mana yang sedang aktif saat scrolling.

**Solusi:**
- Implementasi custom hook `useActiveSection` menggunakan Intersection Observer
- Highlight link aktif dengan underline animasi atau dot indicator
- Smooth transition antar state aktif

**File baru:**
- `hooks/useActiveSection.ts` [NEW]

---

### 1.4 Tombol CTA "Daftar" dengan Micro-interaction
**Masalah:** Tombol "Daftar" terlihat flat dan kurang menarik perhatian.

**Solusi:**
- Tambah subtle gradient pada hover
- Implementasi ripple effect saat click
- Tambah icon kecil (UserPlus) yang muncul saat hover
- Pulse animation ringan untuk menarik perhatian pertama kali

## PHASE 2: HERO SECTION UPGRADE
**Fokus:** Visual Impact, Animasi, dan Konversi

### 2.1 Animated Text dengan Split Character Animation
**Masalah:** Headline muncul dengan animasi sederhana (fade + slide), kurang memorable.

**Solusi:**
- Split headline menjadi per-karakter untuk animasi stagger
- Kata "Profesional" dianimasikan dengan reveal dari bawah per huruf
- Delay berbeda untuk setiap baris teks
- Gunakan `motion.span` untuk setiap karakter

---

### 2.2 Floating Elements / Subtle Pattern Background
**Masalah:** Background terlalu polos, kurang depth visual.

**Solusi:**
- Tambahkan subtle dot grid pattern atau geometric shapes
- Floating elements dengan parallax effect ringan (bergerak saat mouse move)
- Opacity sangat rendah (5-10%) agar tidak mengganggu konten
- Gunakan CSS pseudo-elements atau SVG pattern

---

### 2.3 Stats Counter di Bawah CTA
**Masalah:** Hero section tidak menampilkan social proof angka.

**Solusi:**
- Tambah row stats: "500+ Magang Tersalurkan | 50+ Perusahaan Partner | 98% Puas"
- Animasi count-up saat masuk viewport menggunakan library seperti `react-countup`
- Style minimalis dengan separator dot atau pipe

---

### 2.4 Video Background atau Illustration Option
**Masalah:** Hero section kurang visual interest untuk first impression.

**Solusi:**
- Opsi 1: Tambah ilustrasi SVG custom di samping atau belakang teks
- Opsi 2: Background video loop muted dengan overlay gradient
- Implementasi dengan lazy loading untuk performa
- Fallback ke static image untuk koneksi lambat

---

## PHASE 3: TRUST LOGOS SECTION POLISH
**Fokus:** Autentikitas dan Interaktivitas

### 3.1 Logo Asli Perusahaan dengan Lazy Loading
**Masalah:** Saat ini hanya menampilkan teks nama perusahaan, kurang profesional.

**Solusi:**
- Ganti teks dengan actual logo images (SVG preferred)
- Simpan di `/public/logos/` dengan format: telkom.svg, goto.svg, dst
- Implementasi lazy loading dengan Next.js Image
- Maintain grayscale filter dengan hover ke colored

---

### 3.2 Infinite Scroll Marquee Animation
**Masalah:** Logo statis, kurang dinamis untuk social proof section.

**Solusi:**
- Implementasi CSS-only atau Framer Motion marquee
- Dua baris logo bergerak berlawanan arah (optional)
- Pause on hover untuk accessibility
- Smooth infinite loop tanpa jump

**CSS contoh:**
```css
@keyframes marquee {
  from { transform: translateX(0); }
  to { transform: translateX(-50%); }
}
```

---

### 3.3 Tooltip dengan Info Perusahaan
**Masalah:** Logo tidak memberikan konteks tambahan.

**Solusi:**
- Tambah tooltip saat hover menampilkan: nama perusahaan + jumlah posisi tersedia
- Gunakan Radix UI Tooltip atau implementasi custom
- Animasi fade-in smooth
- Position: bottom-center dengan arrow

---

### 3.4 "Lihat Semua Partner" Link dengan Modal Gallery
**Masalah:** Hanya menampilkan 5 logo, user tidak tahu ada lebih banyak.

**Solusi:**
- Tambah link "Lihat 50+ Partner Lainnya" di bawah logo row
- Click membuka modal dengan grid semua partner logos
- Search/filter functionality dalam modal
- Grouped by industry (Teknologi, BUMN, Startup, dll)

## PHASE 4: PHILOSOPHY SECTION ELEVATION
**Fokus:** Visual Hierarchy dan Engagement

### 4.1 Icon Visual untuk Setiap Feature
**Masalah:** Feature cards hanya menggunakan border-left sebagai visual cue.

**Solusi:**
- Tambah icon dari Lucide untuk setiap feature:
  - Terkurasi Ketat: Shield/CheckCircle
  - Ekosistem Kampus: GraduationCap
  - Jalur Karier: TrendingUp
- Icon dengan background circle subtle
- Color accent pada hover

---

### 4.2 Background Pattern atau Gradient Mesh
**Masalah:** Section ini penting tapi visualnya flat.

**Solusi:**
- Tambah gradient mesh background (seperti vercel.com)
- Atau gunakan organic blob shapes dengan blur
- Opacity rendah 10-15%
- Animate movement subtle dengan CSS

---

### 4.3 Expand/Collapse Detail Content
**Masalah:** Deskripsi feature terlalu pendek, user mungkin ingin tahu lebih.

**Solusi:**
- Tambah "Pelajari Lebih Lanjut" link per feature card
- Click expand menampilkan paragraf tambahan dengan animasi height
- Atau redirect ke page detail terpisah
- Accordion behavior: close other saat expand satu

---

### 4.4 Number Counter untuk Credibility
**Masalah:** Section tidak memiliki data kuantitatif pendukung.

**Solusi:**
- Tambah metrics row: "150+ Lowongan Terkurasi | 30 Kampus Partner | 4.9/5 Rating"
- Posisi: di bawah quote atau di atas feature cards
- Style: subtle, font-weight medium, spacing lebar
- Animated count-up on viewport enter

---

## PHASE 5: PROCESS SECTION TRANSFORMATION
**Fokus:** Visual Flow dan Interaktivitas

### 5.1 Timeline/Connector Visual antar Steps
**Masalah:** 3 steps terlihat independen, tidak menunjukkan flow/progression.

**Solusi:**
- Tambah connecting line (dashed atau solid) antar steps
- Progressively animate line saat scroll
- Dot indicators pada setiap step yang fill saat active
- Responsive: horizontal di desktop, vertical di mobile

---

### 5.2 Ilustrasi/Icon untuk Setiap Step
**Masalah:** Hanya angka serial, kurang visual interest.

**Solusi:**
- Tambah icon atau mini-illustration per step:
  - 01 Eksplorasi: Search/Compass icon
  - 02 Seleksi: Filter/CheckSquare icon
  - 03 Accelerate: Rocket/Zap icon
- Background circle dengan branded color
- Animate icon on hover (scale/rotate subtle)

---

### 5.3 Interactive Step Cards dengan Hover State
**Masalah:** Steps tidak interaktif, terasa seperti static content.

**Solusi:**
- Card elevation on hover dengan shadow
- Expand deskripsi dengan detail tambahan
- Cursor pointer untuk indicate interactivity
- Optional: link ke page detail proses

---

### 5.4 Progress Indicator untuk User Journey
**Masalah:** User tidak tahu posisi mereka dalam journey flow.

**Solusi:**
- Sticky mini-indicator di sidebar atau bottom
- Menunjukkan: "Anda di tahap Eksplorasi"
- Update berdasarkan user state (if authenticated)
- Skip to step functionality

## PHASE 6: JOBS SECTION ENHANCEMENT
**Fokus:** Fungsionalitas, Filter, dan UX

### 6.1 Filter & Sort Functionality
**Masalah:** 6 job cards ditampilkan tanpa opsi filter atau sort.

**Solusi:**
- Tambah filter bar: Work Mode (WFO/Hybrid/Remote), Location, Industry
- Sort by: Terbaru, Populer, Gaji (jika ada)
- Pill/chip style filters dengan clear all option
- Show count: "Menampilkan 6 dari 150 lowongan"

---

### 6.2 Enhanced Job Card dengan Lebih Banyak Info
**Masalah:** Job card hanya menampilkan title, company, location, dan work mode.

**Solusi:**
- Tambah: logo perusahaan (mini), salary range (jika tersedia), deadline
- Tags untuk skills required (React, Python, etc)
- "Baru" badge untuk listing < 3 hari
- Bookmark/save functionality

---

### 6.3 Quick Apply Modal atau Preview Drawer
**Masalah:** "Detail" link mengarah ke "#", tidak ada aksi nyata.

**Solusi:**
- Click "Detail" membuka side drawer dengan full job description
- Include: requirements, responsibilities, benefits, company info
- "Apply Now" CTA dalam drawer
- Share job functionality

---

### 6.4 Pagination atau Infinite Scroll
**Masalah:** Hanya 6 jobs ditampilkan, tidak scalable.

**Solusi:**
- Opsi 1: Load More button dengan pagination
- Opsi 2: Infinite scroll dengan skeleton loading
- Show total count dan current page
- Maintain scroll position saat kembali

## PHASE 10: GLOBAL IMPROVEMENTS
**Fokus:** Performance, Accessibility, dan Polish

### 10.1 Loading States & Skeleton Screens
**Masalah:** Tidak ada feedback visual saat loading.

**Solusi:**
- Skeleton loaders untuk Job Cards, Testimonials
- Suspense boundaries untuk code splitting
- Progress indicator untuk form submissions
- Optimistic UI updates

---

### 10.2 Accessibility (a11y) Audit & Fixes
**Masalah:** Belum optimal untuk screen readers dan keyboard navigation.

**Solusi:**
- Proper heading hierarchy (h1 > h2 > h3)
- ARIA labels untuk semua interactive elements
- Focus visible indicators
- Color contrast checks (WCAG AA minimum)
- Skip to main content link

---

### 10.3 Performance Optimization
**Masalah:** Belum dioptimasi untuk Core Web Vitals.

**Solusi:**
- Lazy load images below the fold
- Font subsetting untuk custom fonts
- Critical CSS inlining
- Component-level code splitting
- Image optimization dengan next/image (sudah ada, verify)

---

### 10.4 Dark Mode Support
**Masalah:** Hanya ada light mode.

**Solusi:**
- CSS variables untuk color theming
- System preference detection (prefers-color-scheme)
- Manual toggle di Navbar
- Persist preference ke localStorage
- Smooth transition antar mode
